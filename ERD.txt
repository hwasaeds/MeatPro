================================================================================
                      3. RELATIONSHIP SUMMARY
================================================================================

  ProductCategory (1) --------< (0..*) Product
    (A category can have many products; a product may belong to one category)

  Product (1) --------< (0..*) ProductionPlan
    (A product can be planned many times; each plan is for one product)

  Product (1) --------< (0..*) ProductionBatch
    (A product can appear in many batches; each batch produces one product)

  ProductionPlan (1) --------< (0..*) WorkOrder
    (A plan can have many work orders; each work order belongs to one plan)

  WorkOrder (1) --------< (0..*) ProductionBatch
    (A work order can have many batches; each batch belongs to one work order)

  Supplier (1) --------< (0..*) PurchaseTransaction
    (A supplier can have many purchases; each purchase is from one supplier)

  ApplicationUser (1) --------< (0..*) AuditLog
    (Referenced by Username string - no formal FK)

  ApplicationUser (1) --------< (0..*) StockMovement
    (Referenced by PerformedBy string - no formal FK)

                      4. ENTITIES WITHOUT FK RELATIONSHIPS

  These entities are standalone (no foreign keys to other domain entities).
  They reference other entities via string fields only:

  Entity              String Reference Fields
  --------            -----------------------
  RawMaterial         SupplierName (logical reference to Supplier.Name)
  FinishedGood        (no references to other entities)
  StockMovement       ItemType, ItemName, PerformedBy, ReferenceNumber
  NotificationItem    (standalone - no references)
  AuditLog            EntityName, EntityId, Username (generic references)
